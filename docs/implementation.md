# 在线课程系统实施说明

## 1. RBAC 边界
- `teacher`：只允许访问 `/teacher/**` 接口，且只能操作自己创建课程下的章节与资源。
- `student`：只允许访问学生学习接口，仅可查看自己已选课程内容。
- JWT 中写入 `uid` 与 `role`，由中间件统一鉴权并注入上下文。

## 2. MinIO 资源流程
1. 教师调用 `init-upload` 获取预签名上传 URL 和 `object_key`。
2. 前端将 `ppt/pdf/txt/video` 直传 MinIO。
3. 上传成功后调用 `confirm` 接口，数据库写入 `object_url`/`object_key`/`mime_type`/`size_bytes`。
4. 学生拉取课程目录后，按章节和资源顺序学习（可直接展示 URL 或二次换签）。

## 3. 分阶段落地
- 阶段1：配置、数据库、JWT、RBAC、MinIO 客户端。
- 阶段2：教师端课程建设链路（课程/章节/资源 + 排序）。
- 阶段3：学生端选课与学习进度。
- 阶段4：缓存和安全增强（Redis 黑名单、目录缓存、限流、审计）。
- 阶段5：自动化测试和压测。

## 4. 测试策略
- 单元测试：JWT 生成/解析、密码哈希、对象键生成、排序参数校验。
- 集成测试：鉴权流程、教师资源上传流程、学生选课和目录拉取流程。
- 回归测试：章节/资源 reorder 的唯一约束、跨角色越权访问。

## 5. 验收标准
- 教师可完整建设课程目录，并调整章节与资源顺序。
- 学生只能学习已选且已发布课程。
- 资源文件落 MinIO，Postgres 仅存对象 URL/Key 与元信息。
- 学习进度可持续更新并支持断点续学。
